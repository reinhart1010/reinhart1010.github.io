Minimax: Penerapan struktur data Heap dalam Combinatorial Game Theory

/posts/2020/10/02/minimax-penerapan-struktur-data-heap-dalam-combinatorial-game-theory.html

Berikut ini adalah jawaban terhadap salah satu tugas kuliah COMP6049 - Algorithm Design and Analysis di BINUS University (Universitas Bina Nusantara).

Salah satu penerapan dari Heap yang sering dipakai dalam Combinatorial Game Theory adalah Minimax Algorithm. Sesuai dengan namanya, algorithm ini memanfaatkan fungsi min() dan max() untuk menentukan langkah apa saja yang harus ditempuh agar seorang maximizing player (pemain yang melangkah yang lebih jauh) dapat memenangkan seorang minimizing player (pemain yang melangkah lebih sedikit). Namun, perlu diketahui bahwa agar struktur data Heap ini dapat dipakai, jumlah kemungkinan langkah atau moves yang dapat dilakukan setiap pemain haruslah 2 (dua).

Pada umumnya, sebuah Combinatorial Game memiliki karakteristik seperti berikut:

  1. Permainan ini hanya terdapat 2 pemain
  2. Permainan ini memiliki sekumpulan state yang memuat posisi kedua pemain (dan state lainnya seperti skor, penalti, dsb.)
  3. Permainan ini memiliki peraturan yang mendefinisikan adanya langkah/move secara legal untuk berpindah antara satu state dengan yang lain. Agar permainan tersebut dinyatakan wajar dan adil, setiap pemain berhak untuk menggunakan langkah-langkah tersebut dalam state apapun.
  4. Para pemain bermain secara bergiliran
  5. Permainan tersebut dilanjutkan hingga mencapai state di mana tidak ada move legal yang dapat digunakan untuk melanjutkan permainan.

Pada umumnya, kita tentu berharap bahwa maximizing player dapat memenangkan permainan tersebut. Dengan demikian, pemain tersebut harus melakukan move yang dapat membuatnya melangkah lebih jauh daripada lawannya. Namun, jika ia tidak berhati-hati, kekalahan justru dapat menimpa pemain tersebut.

Karena itu, algoritma minimax bekerja sebagai berikut.

Pertama, setiap state direpresentasikan sebagai node (dalam Heap), sedangkan setiap move direpresentasikan sebagai arah-arah panah yang menghubungkan suatu node (state) dengan children (state berikutnya). Di dalam heap ini, setiap leaf (node yang tidak memiliki children) merepresentasikan state dimana sudah tidak ada move legal lagi yang dapat dipakai.

Seluruh node dalam tingkat ganjil merupakan state di mana maximizing player dapat melakukan move. Sedangkan, seluruh node dalam tingkat genap merupakan state di mana minimizing player dapat melakukan move. Jika parent dari sebuah leaf merupakan node tingkat ganjil, maka parent tersebut akan mencari nilai terbesar di antara children tersebut. Sebaliknya, jika parent dari sebuah leaf merupakan node tingkat genap, maka parent tersebut akan mencari nilai terkecil di antara children tersebut.

Proses pencarian tersebut terus dilakukan pada parent berikutnya, sampai ke dalam root node. Setelah itu, proses traversal DFS digunakan untuk mencari kombinasi moves yang optimal untuk mengalahkan minimizing player.

Demikian penjelasan dari saya. Terima kasih.
