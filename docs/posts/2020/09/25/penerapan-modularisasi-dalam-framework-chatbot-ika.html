Penerapan Modularisasi dalam framework chatbot Ika

/posts/2020/09/25/penerapan-modularisasi-dalam-framework-chatbot-ika.html

Berikut ini adalah jawaban terhadap salah satu tugas kuliah COMP6056 - Program Design Methods di BINUS University (Universitas Bina Nusantara).

Pada kesempatan kali ini, saya akan menjelaskan contoh penerapan modularisasi dalam salah satu proyek program yang sedang saya kembangkan. Program ini akan ditulis menggunakan bahasa pemrograman TypeScript, yakni salah satu varian dari JavaScript dengan validasi tipe data input dan output secara ketat.

Untuk sementara ini, nama program tersebut adalah Ika (https://github.com/reinhart1010/ika). Program ini dikembangkan secara procedure-driven dengan fungsi menerima dan mengirim pesan sebagai chatbot dalam beragam aplikasi pesan instan. Adapun tujuan dari pengembangan program ini, yakni memudahkan para pengembang (developer) lainnya untuk mengintegrasikan layanan chatbot mereka ke dalam berbagai aplikasi pesan instan (seperti LINE, Telegram, Discord, WhatsApp, serta Facebook Messenger) secara mudah.

Perumusan Masalah

Pertama-tama, perlu diketahui bahwa setiap aplikasi pesan instan memiliki berbagai perbedaan dalam:

  1. proses autentikasi: Menggunakan username dan password? Chatbot ID dan secret token? Atau menggunakan JWT dan OAuth2?
  2. cara menerima pesan baru: Apakah menggunakan sistem polling (seperti IRC dan Telegram) atau webhook?
  3. tipe pesan yang dikirim: Apakah saya bisa mengirim pesan video dan stiker? Apakah saya dapat menggunakan button dalam pesan yang dikirim?
  4. tipe pesan yang diterima: Apakah saya dapat menerima pesan berupa file CSV?

Masalah seperti yang dijelaskan dalam poin 3 dan 4 merupakan masalah yang serius dalam pengembangan chatbot pada umumnya. Aplikasi LINE, misalnya, dapat menerima berbagai format pesan dari teks sederhana hingga tipe pesan kompleks seperti Carousel dan Flex Message (isi dan layout pesan diatur menggunakan CSS flexbox). Sedangkan, aplikasi lainnya seperti WhatsApp hanya dapat menerima format pesan yang terbatas, yakni teks, gambar, audio, video, stiker, lokasi, serta file/document. Karena itu, setiap pesan yang akan dikirim dapat terdiri atas beberapa format, yang akan nanti dipilih untuk setiap aplikasi pesan instan yang ada.

Salah satu prototipe Object/record pesan teks yang sedang dikembangkan

Tahap Modularisasi (Modul Pengiriman Pesan)

Karena itu, untuk mengirim sebuah pesan, program ini akan melakukan proses seperti berikut. Dalam table di bawah, message_data merupakan record (setara dengan struct dalam C/C++ dan Object di JavaScript) pesan yang akan dikirim, seperti contoh di atas. user_data juga merupakan sebuah record yang berisi identitas user seperti group/chatroom ID, user ID, nomor telepon, email, dan sebagainya.

InputProcessOutput
message_dataValidate inputsfinal_message
application_nameSelect the preferred message format according to application_nameerror_response
user_dataSend the message to user_data via application_name

Berdasarkan data di atas, program ini kemudian akan dibagi menjadi setidaknya 4 modul, yang akan diterapkan menggunakan konsep OOP dan polymorphism yang serupa dengan program youtube-dl.

  • Sebuah modul untuk menghubungkan program terhadap API masing-masing aplikasi pesan instan (interface Provider)
  • Sebuah modul untuk mendefinisikan record jenis dan isi pesan yang ditujukan terhadap user tertentu (interface MessageObject dan MessageContent)
  • Sebuah modul untuk melakukan validasi terhadap format input program tersebut (class Main serta interface Provider)
  • Sebuah modul untuk mengolah isi pesan menjadi format yang diinginkan oleh masing-masing aplikasi (interface Parser, yang akan nanti diimplementasikan di dalam para class yang menganut interface Provider)

Di sini, proses parsing dan komunikasi untuk setiap aplikasi diatur dalam satu interface yang sama: Provider. Hal ini memang berbeda dengan mekanisme yang dianut oleh program youtube-dl (yakni memisahkan kedua hal tersebut menjadi InfoExtractor serta FileDownloader) karena format payload API yang dikembangkan oleh setiap aplikasi pesan instan tersebut berbeda-beda.

Dengan adanya modularisasi, program ini dapat dikembangkan lebih lanjut melalui mekanisme extension, baik untuk menambahkan Provider baru serta menyediakan polyfill Parser terhadap format pesan yang belum didukung oleh sebagian aplikasi pesan instan. Misalnya, jika user hendak mengirim sebuah dokumen PDF, sedangkan aplikasi yang dituju belum dapat menerima format file tersebut, sebuah modul polyfill dapat dijalankan untuk meng-upload file kepada web server untuk sementara, kemudian membagikan URL download dalam bentuk pesan teks kepada penerima dalam aplikasi tersebut.
